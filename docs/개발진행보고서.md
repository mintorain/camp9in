## 4. 데이터베이스 스키마

### 테이블 관계도
```
applicants (1) ──┬── (N) applicant_schools
                 └── (N) applicant_subjects

settings (key-value)
closed_subjects (과목 마감 관리)
page_views (접속 분석)
```

### 주요 컬럼
| 테이블 | 핵심 컬럼 | 용도 |
|--------|----------|------|
| applicants | status, confirmed_subject | 선발 상태 + 확정 과목 |
| applicant_schools | applicant_id, school_id | 지원 학교 매핑 |
| applicant_subjects | applicant_id, subject_id | 지원 과목 매핑 (순서=우선순위) |
| settings | show_counts, show_status, admin_password | 공개 설정 |
| page_views | page, session_id, ip | 접속 분석 |

## 5. API 엔드포인트

| Method | Path | 인증 | 기능 |
|--------|------|------|------|
| POST | /api/applicants | - | 지원서 접수 |

| PATCH | /api/applicants/[id] | Admin | 상태/확정과목 변경 |
| DELETE | /api/applicants/[id] | Admin | 지원자 삭제 |
| GET | /api/applicants/counts | - | 과목별 지원 수 |
| GET | /api/applicants/export | Admin | CSV 내보내기 |
| POST | /api/applicants/status | - | 합격 조회 |
| POST | /api/auth | - | 로그인/비밀번호 변경 |
| GET/POST | /api/analytics | Admin/- | 접속 통계 |
| GET/POST | /api/settings | Admin/- | 공개 설정 |
| GET/POST | /api/subjects/close | Admin/- | 마감 관리 |
| GET | /api/version | - | 배포 버전 체크 |
