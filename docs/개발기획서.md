# Camp9in 개발 기획서
## AI로 만나는 미래 교육 체험 캠프 - 강사 모집 플랫폼

> 버전: 1.0
> 작성일: 2026-04-09
> 주관: 두온교육(주)

## 1. 프로젝트 배경 및 목적

### 1.1 배경
두온교육(주)은 초등학교를 대상으로 AI·미래기술 체험 캠프를 운영하고 있습니다.
2026년 4~6월 3개 초등학교에서 캠프를 진행하기 위해, 각 체험 분야별 전문 강사를 모집해야 합니다.

### 1.2 목적
- 강사 모집 과정의 온라인화 및 자동화
- 지원서 접수부터 선발까지 일원화된 관리 시스템 구축
- 지원자에게 투명한 진행 상태 공개

### 1.3 이해관계자
| 역할 | 대상 | 요구사항 |
|------|------|---------|
| 운영자 | 두온교육(주) 캠프사업부 | 지원자 관리, 선발, 배정 |
| 지원자 | 외부 강사 (교육 경력자) | 지원서 작성, 결과 조회 |
| 학교 | 미금초, 정림초, 청원초 | 일정 확인, 강사 명단 |

## 2. 서비스 구성

### 2.1 서비스 맵

```
camp9in.duonedu.net
├── / ......................... 메인 랜딩 페이지
├── /apply ................... 강사 지원서 작성 (4단계)
├── /apply/complete .......... 지원 완료 확인
├── /status .................. 합격 여부 조회
├── /admin ................... 관리자 로그인
├── /admin/dashboard ......... 관리자 대시보드
├── /admin/applicants ........ 지원자 목록 관리
└── /admin/assigned .......... 배정 강사 명단
```

### 2.2 사용자 흐름

#### 지원자 흐름
```
메인 페이지 방문
    ↓
캠프 정보 확인 (학교, 과목, 일정, 강사료)
    ↓
"강사 지원하기" 클릭
    ↓
Step 1: 기본 정보 입력 (이름, 연락처, 이메일, 생년월일, 주소)
    ↓
Step 2: 학교 선택 → 학교별 과목 선택
    ↓
Step 3: 학력, 경력, 자격, 자기소개 입력
    ↓
Step 4: 강사료 확인 → 개인정보 동의 → 제출
    ↓
지원 완료 페이지
    ↓
(이후) 합격 조회 페이지에서 결과 확인
```

#### 관리자 흐름
```
관리자 로그인 (/admin)
    ↓
대시보드 확인
├── 접속자 현황 (실시간, 일별, 누적)
├── 지원자 요약 (전체, 학교별, 상태별)
├── 과목별 지원 현황 + 마감 관리
├── 학교별 과목 지원 현황
├── 공개 설정 (인원수, 합격조회)
└── 빠른 이동 (지원자 목록, 배정 명단)
    ↓
지원자 목록 관리
├── 검색 (이름, 연락처, 이메일)
├── 필터 (학교, 과목, 상태, 중복)
├── 상세 조회 (전체 지원 내용)
├── 상태 변경 (접수 → 검토 → 합격/불합격)
├── 확정 과목 지정
└── 지원자 삭제
    ↓
배정 강사 명단
└── 학교별 → 과목별 합격 강사 조회
```

## 4. 기능 상세 명세

### 4.1 메인 랜딩 페이지 (/)
- 3D 스크롤 히어로 애니메이션
- 학교별 모집 현황 카드 (체험 분야 수, 분야별 모집 인원)
- 캠프 소개 (운영 목표, 부스 순환, 강사 역할)
- 학교별 일정 카드 (3D 틸트 효과, 학년별 과목 표시)
- 모집 분야 그리드 (과목별 카드 + 마감/지원 수 표시 + 학교 태그)
- 지원 절차 (01→02→03 스텝)
- CTA (지원하기 버튼)
- 헤더 네비게이션 (캠프소개/일정/모집분야/지원절차)
- 결과 조회 링크 (관리자 공개 시만 표시)

### 4.2 지원서 작성 (/apply)
**4단계 스텝 폼:**

| 단계 | 필드 | 필수 |
|------|------|:----:|
| Step 1 | 이름, 연락처, 이메일, 생년월일, 주소 | O |
| Step 2 | 지원 학교 (복수), 학교별 과목 (각 1개) | O |
| Step 3 | 최종 학력, 전공, 경력, 자격, 자기소개 | 학력·경력만 필수 |
| Step 4 | 강사료 확인, 개인정보 동의 | 동의 필수 |

**유효성 검증:**
- 이름: 2~10자 한글
- 연락처: 010-XXXX-XXXX 자동 포맷
- 이메일: 이메일 형식
- 주소: 3~100자
- 경력: 20자 이상
- 자기소개: 500자 이하
- 학교별 과목: 동시 운영이므로 학교당 1개만

### 4.3 합격 조회 (/status)
- 이름 + 연락처 입력으로 조회
- 상태별 시각적 피드백:
  - 접수완료 (파란색)
  - 검토중 (노란색)
  - 합격 (초록색)
  - 불합격 (빨간색)
- 지원 학교, 과목, 접수일 표시
- 관리자 비공개 설정 시 "아직 결과를 조회할 수 없습니다" 표시

### 4.4 관리자 대시보드 (/admin/dashboard)
**사이드바 네비게이션:** 대시보드 / 지원자 목록 / 배정 강사 명단

**대시보드 구성:**
- 실시간 접속 현황 (4개 카드: 실시간, 오늘, 누적, 지원서 조회)
- 지원 요약 (전체, 학교별, 선발 완료)
- 상태별 현황 + 최근 7일 방문 통계
- 과목별 지원 현황 + 마감 토글
- 학교별 과목 지원 현황 (학년별 구분)
- 설정 패널 (인원수 공개, 합격조회 공개, 비밀번호 변경)

### 4.5 지원자 목록 (/admin/applicants)
- 학교별·과목별 지원 현황 요약 (상단)
- 검색: 이름, 연락처, 이메일
- 필터: 학교, 과목, 상태, 중복 지원자
- 테이블: 이름, 연락처, 학교, 과목(확정 표시), 상태, 지원일시
- 상세 모달: 전체 지원 내용 + 상태 변경 + 확정 과목 선택 + 삭제

### 4.6 배정 강사 명단 (/admin/assigned)
- 합격(선발) 상태 지원자만 표시
- 학교별 → 과목별 테이블
- 확정 과목 기준 우선 배정 (미확정 시 지원 과목 기준)
- 강사명, 연락처, 이메일 표시

## 5. 관리자 제어 기능

### 5.1 공개 설정
| 설정 | 키 | 기본값 | 영향 범위 |
|------|-----|--------|----------|
| 인원수 공개 | show_counts | true | 메인 과목 카드, 지원서 과목 선택 |
| 합격조회 공개 | show_status | false | /status 페이지, 헤더 "결과 조회" 링크 |

### 5.2 과목 마감
- 수동 마감/해제: 관리자가 직접 토글
- DB `closed_subjects` 테이블에서 관리
- 마감된 과목은 지원서에서 선택 불가

### 5.3 지원자 상태 흐름
```
접수완료(pending) → 검토중(reviewing) → 합격(accepted)
                                      → 불합격(rejected)
```

### 5.4 확정 과목
- 합격 처리 시 `confirmed_subject`로 최종 과목 지정
- 여러 과목 지원자의 최종 배정 과목 결정
- 배정 명단에서 확정 과목 기준으로 표시

## 6. 비기능 요구사항

### 6.1 성능
- 페이지 로드: 3초 이내
- API 응답: 1초 이내
- 30초마다 실시간 접속자 갱신
- 30초마다 배포 버전 체크 (자동 새로고침)

### 6.2 호환성
- 모바일 반응형 (320px ~ 1920px)
- 주요 브라우저: Chrome, Safari, Edge, Samsung Internet
- 관리자 페이지: 데스크톱 사이드바 + 모바일 헤더

### 6.3 접근성
- WCAG 2.2 Level AA 대응
- 시맨틱 HTML (header, nav, main, footer, section)
- ARIA 속성 (role, aria-label, aria-modal)
- 키보드 네비게이션 지원
- 포커스 표시

### 6.4 보안
- 관리자 인증 (Bearer Token)
- 입력값 서버 검증 (Zod)
- SQL Parameterized Query (인젝션 방지)
- 개인정보 수집 동의 필수
- 보유 기간: 모집 완료 후 1년
