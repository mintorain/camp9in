## 4. 랜딩페이지 구성

### 4-1. 페이지 구조

```
[히어로 섹션]
  - 캠프 타이틀 + 강사 모집 안내
  - CTA 버튼: "강사 지원하기"

[캠프 소개 섹션]
  - 캠프 목표 및 운영 방식 (부스 순환 체험)
  - 교육 목표: AI 시대 창의력·문제해결력 키우기

[일정 및 장소 섹션]
  - 정림초 / 청원초 일정 카드
  - 각 학교별 날짜, 시간, 장소, 모집인원

[모집 분야 섹션]
  - 8개 분야 카드 그리드
  - 각 분야명, 활동 내용, 필요 역량 표시

[강사 혜택 섹션]
  - 강사비 안내
  - 활동 인증서 발급
  - 경력 증명서 제공

[지원 절차 안내]
  - STEP 1: 지원서 작성
  - STEP 2: 서류 검토
  - STEP 3: 최종 선발 통보

[지원서 작성 폼 섹션]
  - (아래 5항 상세 참조)

[푸터]
  - 운영사 정보, 문의처
```

## 10. 사용자 플로우

### 10-1. 강사 지원 플로우

```
랜딩페이지 방문
  → "강사 지원하기" CTA 클릭
  → 지원서 작성 폼
  → 학교/과목 선택
  → 개인정보 입력
  → 경력/자격 입력
  → 개인정보 동의 체크
  → 제출
  → 지원 완료 페이지 (접수번호 안내)
  → 확인 이메일 발송 (선택)
```

### 10-2. 관리자 플로우

```
관리자 로그인
  → 대시보드 (전체 현황 확인)
  → 학교별 필터로 지원자 확인
  → 과목별 필터로 지원자 확인
  → 개별 지원자 상세 보기
  → 상태 변경 (접수완료 → 검토중 → 선발/불합격)
  → CSV 내보내기
```

## 12. 접근성 요구사항

- 모든 폼 요소에 `<label>` 연결
- 키보드만으로 전체 지원 프로세스 완료 가능
- 색상 대비 4.5:1 이상 준수
- 스크린 리더 호환 (aria-label, aria-live)
- 에러 메시지 명확한 안내 (어떤 필드가 왜 잘못됐는지)
- `prefers-reduced-motion` 지원

## 13. 보안 요구사항

- 개인정보 암호화 저장 (연락처, 이메일)
- HTTPS 전송 필수
- SQL 인젝션 방지 (Supabase 파라미터 바인딩)
- XSS 방지 (입력값 이스케이프)
- 관리자 페이지 인증 필수
- CSRF 토큰 적용
- Rate limiting (지원서 제출: 분당 5회 제한)
